# Project Constitution

This document governs how this project is built and how any AI assistant working on it must behave.

1. Evidence before conclusions — every claim must trace back to a real data source.
2. Explainability over complexity — every score must be understandable in plain English.
3. Modular architecture — data, scoring, and UI stay separate and swappable.
4. Professional documentation — every feature gets a documentation update.
5. Incremental weekly progress — every week ships something visibly working.
6. Small, testable features — no giant untested leaps.
7. Free tools whenever practical.
8. Long-term maintainability over cleverness.
9. Never fabricate financial information.
10. Always distinguish facts from opinions.
